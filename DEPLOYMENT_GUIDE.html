📋 Deployment Steps
Follow these 5 steps in order. Takes about 10 minutes total.
1
👤 Create a Free GitHub Account
GitHub is a free platform for hosting websites and code. You need a free account to deploy your app.
  • 1Open your browser and go to https://github.com
  • 2Click the "Sign up" button in the top-right corner
  • 3Enter your email address → Click Continue
  • 4Create a password → Click Continue
  • 5Choose a username (e.g. ortholite-india or your name) → Click Continue
  • 6Solve the verification puzzle → Click Create account
  • 7Check your email inbox → Click the verification link GitHub sent you
✅Your GitHub account is now ready. You can skip this step if you already have an account.
2
📁 Create a New Repository
A "repository" is like a folder on GitHub where your app files will live. The repository name becomes part of your URL.
  • 1After logging in, click the "+" icon in the top-right corner
  • 2Click "New repository" from the dropdown menu
  • 3In "Repository name" field type exactly: ortholite-qms
  • 4In "Description" type: Ortholite India Quality Management System
  • 5Under visibility, select "Public" — this is required for free GitHub Pages hosting
  • 6Scroll down and check "Add a README file"
  • 7Click the green "Create repository" button at the bottom
💡Important: The repository must be set to Public. Private repositories require a paid plan for GitHub Pages.
3
📤 Upload All App Files
Now upload all the QMS app files into your repository. First extract the ZIP file you downloaded.
  • 1Find the downloaded ZIP file: Ortholite_QMS_WebApp.zip → Right-click → Extract All
  • 2Open the extracted folder → Open the qms-app folder inside it
  • 3In your GitHub repository page, click "Add file" → "Upload files"
  • 4Select ALL the files and folders shown below and drag them into the upload area
  • 5Wait for all files to finish uploading (green checkmarks appear)
  • 6In the "Commit changes" section at the bottom, type: Initial QMS deployment
  • 7Click the green "Commit changes" button
FILES TO UPLOAD (select all of these):
📄 index.html
📄 app.html
📄 404.html
📄 .nojekyll
📁 css/ (entire folder)
📁 js/ (entire folder)
📁 assets/ (entire folder)
📄 README.md
⚠️The .nojekyll file is hidden (starts with a dot). On Windows, make sure "Show hidden files" is enabled in File Explorer to see and select it. On Mac, press Cmd+Shift+. to show hidden files.
4
⚙️ Enable GitHub Pages
Tell GitHub to publish your repository as a live website. This takes just a few clicks.
  • 1In your repository, click the "Settings" tab (in the top navigation bar)
  • 2In the left sidebar, scroll down to find "Pages" and click it
  • 3Under "Source", click the dropdown and select "Deploy from a branch"
  • 4Under "Branch", select "main" from the first dropdown
  • 5In the second dropdown (folder), select "/ (root)"
  • 6Click the "Save" button
  • 7A blue/green banner appears with your site URL — note it down
  • 8Wait 2 to 5 minutes for GitHub to build and publish your site

✅GitHub will show a green banner: "Your site is live at https://YOUR-USERNAME.github.io/ortholite-qms/"
✓
🎉 Access Your Live QMS App
Your app is now live on the internet! Open the URL on any device — computer, phone, or tablet.
  • 1Open Google Chrome on any device
  • 2Type your URL: https://YOUR-USERNAME.github.io/ortholite-qms/
  • 3The Ortholite QMS login screen will appear
  • 4Login with your credentials (see below)
  • 5Share the URL with your team members — they can access from any device!

⚠️ Important Notes

  • Data is saved in browser localStorage — each device has its own data
  • Use Excel Export/Import to transfer data between devices
  • Repository must stay Public for free hosting
  • To update the app — re-upload changed files to GitHub
  • Anyone with the URL can see the login page (data is password protected)
  • Works on Chrome, Firefox, Edge, Safari

🔄 How to Update the App Later

  • Receive updated ZIP file
  • Extract and open qms-app folder
  • Go to your GitHub repository
  • Click Add file → Upload files
  • Upload the updated files (they overwrite old ones)
  • Commit changes — live in 2–5 minutes
